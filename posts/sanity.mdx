## Sanity

Sanity는 Headless CMS 종류 중 하나 입니다.

-   **Content Lake** 라는 cloud 상에 존재하는 데이터 보관소 입니다.(cloud형 DB)
-   Content Lake 에 접근을 하기 위해 **Sanity Studio**를 사용 합니다.
-   Front에서는 Sanity 에서 제공하는 API를 통해서 Content Lake에 접근하고 데이터를 읽고, 사용 할 수 있습니다.

하지만 Next.js를 사용 할 예정이기 때문에 Next에서 가지고 있는 서버로 Content Lake에 접근하고 <br/> Front는 서버에 요청을 하는 식으로 해보겠습니다.

![](https://github.com/Choi-HyunHo/blog/assets/87301268/caec89fa-cffa-4fbf-8509-f8e3883c9415)

<br />

### Front에서 직접 Content Lake에 접근하지 않고 Server를 통해 접근하는 이유

1. 유효성 검사를 할 수 있기 때문에
2. 접근에 대한 권한이 있는 사용자인지

데이터를 직접 접근 하는 것보다 보안을 조금 더 생각 할 수 있고 <br/> 현업에서도 DB에 프론트가 직접 접근하지 않고 백엔드를 통해서 접근하는 경우가 더 많기 때문에 <br/> 공부가 될거라고 생각 합니다.
